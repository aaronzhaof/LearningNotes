冒泡排序

冒泡排序的基本语法

						var a = [4,1,2,5,6];
						  function sortData(){
								for(var i=0; i < a.length;i++){
									for(var j=i+1;j <= a.length;j++){
										if(i > j){
											var temp=i;
												i=j;
												j=temp;
										}
									}
								}
						  }

--在学习JavaScript DOM 编程艺术一书练习中发现一个错误，有一个项目中要在CSS中的 background-image属性中调用一张图片，发现无论如何修改，总是调用失败。路径无误，语法无误。


最后经检查，原因是我在另一CSSCSS文件中设置背景色时，将background-color简写成background ,在这之前我一直这么用也没发现问题。今天才发现，用了 background属性后，再想精确设置background 属性中的其他子属性，如 background-color,background-position等属性时，他们的值会完全被background 属性的值所覆盖。所以在网页中显示不出来。


嗯，所以说人不能偷懒啊！

最近发现因为迷之原因，电脑上面的chrome浏览器和IE浏览器启动的时候，还有本地html文件在他们上面打开的时候，都会被强制跳转到搜狗主页 http://www.543891.com/。 在网上查了各种方法，快捷方式目标栏也没有被修改，用360杀毒好像第一次 查出来了，但是杀完了依旧没什么卵用，病毒还是继续猖狂。后面360也查不出来了，按照网上说的最后一个办法，删除快捷方式，然后在chrome的源目录 修改浏览器的名称，再把修改后的浏览器快捷方式发送到桌面，然后把快捷方式拖到任务栏固定，这样就没什么问题了。本地html打开也没什么问题了。

Node.removeChild()


node.removeChild()方法从DOM中删除一个子节点


						var oldChild = node.removeChild(child);

						//OR

						element.removeChild(child);

  • child是要移除的那个子节点。
  • node是child的父节点
  • oldChild保存对删除子节点的引用。oldChild===child.

被移除的这个子节点仍然存在于内存中，只是没有添加到当前文档的DOM树中， 因此，你还可以把这个节点重新添加回文档中，当然，实现要用另外一个变量比如上 例中的 oldChild来保存这个节点的引用。如果使用上述语法中的第 二种方法，即没有使用oldChild来保存对这个节点的引用，则认为被移除的节点已 经是无用的，在短时间内将会被内存管理回收。


如果上例中的child节点不是node节点的子节点，则该方法会抛出异常。
